## Avant de toucher l'historique

- [ ] `git status` propre ou maîtrisé
- [ ] Sauvegarde de sécurité (branche, stash, ou tag)
- [ ] Objectif clair de l'opération

## Avant push

- [ ] Message de commit explicite
- [ ] Historique lisible
- [ ] Tests essentiels passés

## Si incident

- [ ] Stop et observer (`status`, `log`, `reflog`)
- [ ] Choisir entre `revert` (partage) et `reset` (local)
- [ ] Revalider après correction
